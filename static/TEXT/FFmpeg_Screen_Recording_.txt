The error you are seeing is likely because the `-cfr` option is not recognized by ffmpeg. The correct option to use for specifying a constant frame rate is `-r`.

So you should replace `-cfr 26` with `-r 24` (or whatever constant frame rate you want to use).

Here's the corrected command:

```
ffmpeg -f x11grab -framerate 24 -video_size 1366x768 -i :0.0 -f pulse -i default -c:v libx264 -preset fast -r 24 -maxrate 2500k -bufsize 5000k -g 60 -vf format=yuv420p -c:a aac -b:a 128k -f flv rtmp://a.rtmp.youtube.com/live2/xxxxxxxxxx
``` 

This command will capture the screen of the display `:0.0`, using the x11grab input format. It will also capture audio from the default pulseaudio input device. The video will be encoded using the libx264 codec, with a preset of `fast`, a constant frame rate of 24 frames per second, a maximum bitrate of 2500k, and a buffer size of 5000k. The video will have a keyframe interval of 60 frames and will be converted to the yuv420p pixel format. The audio will be encoded using the aac codec with a bitrate of 128k. Finally, the output will be sent to an RTMP server at the specified URL.
